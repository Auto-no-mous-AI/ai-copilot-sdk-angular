# Security Policy

## Supported Versions

The main branch and the latest tagged production release are supported for security fixes.
Older local demo, playground, and prototype snapshots are not supported unless a customer agreement says otherwise.

## Reporting A Vulnerability

Report suspected vulnerabilities privately to the repository owner or the Auto-no-mous AI security contact before opening a public issue.
Include the affected repository, commit SHA, environment, reproduction steps, and any known customer or data exposure.

## Response Expectations

- Critical: acknowledge within 1 business day and start remediation immediately.
- High: acknowledge within 2 business days and prioritize a fix in the next release window.
- Medium/Low: triage during normal planning and document the accepted risk or fix path.

## Handling Rules

Do not commit secrets, customer data, access tokens, production logs with PII, or exploit payloads to this repository.
Use synthetic data in tests, examples, screenshots, and documentation.
